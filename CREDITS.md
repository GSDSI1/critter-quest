## Art

### Procedural / generated

Most critter sprites, UI chrome, and the default tileset are generated at build time by scripts in `scripts/` (`npm run gen-assets`).

### Hand pixel art

Species overrides in `scripts/critter-art/` (starters, batch5–6, shapelib body plans).

### Shape library (P14/P20)

Non-bespoke species use `scripts/critter-art/shapelib.mjs` — nine body plans (blob, quadruped, serpent, avian, humanoid, crystalline, insect, aquatic, armored) with distinct backs, blink frames, and per-species accents.

### Kenney — Tiny Town (CC0)

Overworld base tiles (grass, path, water, trees, building accents) may be sourced from [Kenney Tiny Town](https://kenney.nl/assets/tiny-town) via `npm run fetch-kenney`.

- **License:** [CC0 1.0 Universal](https://creativecommons.org/publicdomain/zero/1.0/)
- **Attribution:** Not required; appreciated — "Kenney.nl" or [kenney.nl](https://kenney.nl)

## Audio

SFX and chiptune BGM loops in `public/assets/audio/` are procedurally generated by `scripts/generate-audio.mjs` (3-channel chiptune, six themes: town, overworld, battle, gym, cave, victory). Loops are **CC0** (self-authored, no third-party samples).

Optional third-party CC0 replacement: drop licensed WAV/OGG into `public/assets/audio/cc0-import/` and map themes in `manifest.json`, then run `npm run import-cc0-bgm` (also runs at end of `gen-assets`). Kenney.nl and OpenGameArt CC0 packs are suitable sources — add attribution in this file when used.

## Fonts

In-game UI uses system / bundled pixel fonts as configured in `src/ui/theme.ts`.
